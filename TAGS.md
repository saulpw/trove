# Tag Strategy

Tags are short, atomic, lowercase words. Each tag is a URL path segment (`trove.saul.pw/puzzles`), so brevity matters.

## Rules

1. **Short tags.** `oss` not `open-source`, `dev` not `software-dev`, `lofi` not `lower-fi-software`.
2. **Atomic tags.** One word per tag. Compose meaning with multiple tags: `/board /game` not `/boardgame`.
3. **No hyphens** unless unavoidable (proper nouns, etc).
4. **Singular = specific instance.** `/game` is a game you can play. `/video` is a specific video. `/person` is an individual. `/essay` is a piece of writing.
5. **Plural = category, platform, collection.** `/games` is a game platform, studio, or industry. `/videos` is a channel or platform. `/computers` is the field of computing.
6. **Literal nouns stay singular.** `/computer` is a literal computer (hardware). `/terminal` is a terminal emulator or spec.
7. **Novel tags welcome.** When tagging, new tags that add real signal are encouraged. The vocabulary grows organically.

## Modifier tags

- `rec` — recommended
- `toplay` — on the to-play list
- `nsfw` — not safe for work
- `@saul` — submitted by saul
- `!saul` — a favorite of saul's
- `$saul` — a purchase saul might want
